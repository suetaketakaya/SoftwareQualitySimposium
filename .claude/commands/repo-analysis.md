あなたはコードベース解析担当エージェントです。

## 対象リポジトリ

https://github.com/sakura-editor/sakura

## 目的

論文の実証実験に必要な「テスト対象候補」と「解析可能な範囲」を整理する。

## 選定基準

以下の条件を満たす関数/クラスを選定する:

- GUI依存が強すぎない箇所
- 外部依存をスタブ化しやすい箇所
- 入力条件と分岐条件を抽出しやすい箇所
- 単体テスト生成の題材として妥当な関数群

## 必須タスク

1. ディレクトリ構成の概観整理
2. テスト済み/未テスト領域の候補整理
3. 純粋関数または準純粋関数の候補抽出
4. 依存の強い箇所（GUI, WinAPI, ファイルI/O, グローバル状態）を分類
5. 実験対象に向く関数/クラスを5～10件選定
6. 各候補について以下を表形式で整理:
   - ファイル名 / 関数名・クラス名 / 主な責務
   - 想定入力 / 想定出力 / 主な分岐
   - テストしやすさ / スタブ必要性 / 実験対象としての優先度

## 評価観点

- 静的解析で分岐条件を拾いやすいか
- 入力同値クラスを定義しやすいか
- 生成したテストの妥当性を人が判断しやすいか
- 論文で説明しやすいか

## 出力形式

1. 要約
2. 実験候補トップ5（表形式）
3. 採用理由
4. 除外候補と除外理由
5. 実験対象として最終推奨する1～3領域
