## SQiP 2026 論文投稿プロジェクト

ソフトウェア品質シンポジウム2026（SQiP 2026）への論文投稿を目的としたリサーチリポジトリです。  
公開コードベース（sakura-editor/sakura）を対象に、**静的解析と LLM を組み合わせてホワイトボックステスト要求モデルを生成し、単体テスト生成・実行・評価までを半自動化する方法**を検討・実証します。

### 研究の背景と狙い

- **背景**: LLM に単体テストを書かせる研究・事例は増えているが、生成根拠が不透明で、再現性や説明可能性に課題がある。
- **狙い**: LLM に直接テストを書かせるのではなく、静的解析で抽出したホワイトボックステスト要求を「人間がレビュー可能な中間成果物」として固定し、その要求に基づいてテスト生成・実行・評価を繰り返すワークフローを提案・評価する。

### ディレクトリ構成（概要）

このリポジトリでは、研究プロセスを次のように分割して管理します。

- `CLAUDE.md`: エージェント向けのプロジェクト全体指示
- `.claude/`: マルチエージェント用コマンド定義
- `knowledge/`: 研究方針・設計ノート
- `analysis/`: 対象リポジトリ（sakura-editor）の解析結果
- `test-requirements/`: ホワイトボックステスト要求モデル
- `generated-tests/`: 生成されたテストコード
- `experiments/`: 実験設定・実行結果
- `drafts/`: 論文草稿・図版
- `report/`: SQiP 提出用テンプレートと最終投稿物

詳細は `CLAUDE.md` と `knowledge/direction.md` を参照してください。

### 実証対象

- OSS エディタ `sakura-editor/sakura`  
  （GitHub: `https://github.com/sakura-editor/sakura`）

主に、GUI 依存が比較的弱く、静的解析とテスト自動化の題材として扱いやすいコンポーネントを対象にします。

### 再現に必要なもの

現時点では、このリポジトリは主に **ドキュメント・設定・生成物** を管理しており、ビルドスクリプトや環境構築手順は今後整理予定です。

想定している環境は以下の通りです（暫定）:

- OS: Windows / macOS のいずれか
- 静的解析ツール: C++ 用解析ツール（検討中）
- LLM 実行環境: Claude Code / GPTs Codex 等

実際の実験手順・依存ツールが固まり次第、`experiments/` 配下または別セクションに詳細を追加します。

### 匿名査読に関する注意

このリポジトリ内のドキュメント（特に `drafts/`, `report/`）では、

- 著者名・所属・個人が特定される情報を本文やファイル名に含めない

という SQiP 2026 の匿名査読要件に従います。

### ライセンス / 取り扱い

- このリポジトリ自体は研究用メタリポジトリです。
- 実証対象である `sakura-editor/sakura` は、各自で公式リポジトリから取得し、そのライセンスに従って利用してください。
